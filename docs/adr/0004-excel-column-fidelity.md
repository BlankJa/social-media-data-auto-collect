# ADR-0004: Excel 列保真——沿用旧表头与命名

## Status
Accepted (2026-06-08)

## Context
此前的工作流靠某 GUI 工具手动导出 Excel，表头里夹着工具痕迹列（如某些「页面源码」「详情链接」等冗余列），文件按 `*-YYYY.MM.DD.xlsx` 风格手工命名归档。下游（人和已有归档）都按这套列序和命名在用。

## Decision
渲染时**照搬旧表头的列序与列名**，工具痕迹列即使内容为空也保留；输出文件名沿用旧命名风格（`B站UP主主页视频采集-YYYY.MM.DD.xlsx` 等）。每平台的列由 `rendering.py` 的 `ColumnSpec` 列表显式定义（B 站 9 列 / 抖音 13 列 / 微博 10 列 / 快手 8 列），并有列表头测试锁定。

## Consequences
- ＋ 新产出与历史归档无缝衔接，下游零改动。
- ＋ 列定义集中可测，改一列有测试兜底。
- － 带着一些没用的旧列。将来要清理时，标准动作是：先和下游确认，再改 `ColumnSpec` + 对应列测试，作为一次显式的破坏性变更。

## Alternatives considered
- **干净重设计列序**：去掉工具痕迹列、重排。放弃——破坏下游兼容，得不偿失；保真优先。
